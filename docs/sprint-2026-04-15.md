# Sprint Report — April 15, 2026

## Infrastructure

### Domain Cutover
Custom domain `imperfectgg.com` was registered 6 days prior but never attached to serve auth/SEO correctly. Users saw `imperfect-sage.vercel.app` in every OAuth redirect, canonical URL, and Team Hub login flow.

- Updated `NEXTAUTH_URL` and `AUTH_URL` prod env vars in Vercel (initial attempt had a literal `\n` in the value from a shell pipe — caught via `/api/auth/providers` introspection)
- Find-and-replaced `imperfect-sage.vercel.app` across `src/app/[locale]/**/layout.tsx` (8 files), `Makefile`, `docs/observability.md`
- Coordinated Discord OAuth redirect URI update in the Developer Portal (user-facing step walked through manually)
- Verified providers endpoint returns correct callback URL post-deploy

### Cron Migration
Vercel Hobby no longer permits sub-daily crons. The `*/5 * * * *` scrim reminders cron was blocking every deploy.

- Removed `crons` block from `vercel.json`
- New `.github/workflows/reminders-cron.yml` hitting `/api/cron/reminders` every 5 minutes
- Added `CRON_SECRET` to GitHub Actions secrets (pulled from Vercel prod env)
- Manual `workflow_dispatch` trigger verified green

### CI Recovery (8 days red)
Root causes:
- `package-lock.json` pinned `@swc/helpers@0.5.15` but Next.js 16.2.1 required `>=0.5.17` — local `npm 11` was lenient, CI's `npm 10` wasn't
- React Compiler's new `set-state-in-effect` rule fired on two initial-load `useEffect` patterns
- Two `any` type errors in the Marvel Rivals API transformer
- API routes migrated to `resolveOrgRole()` (uppercase `orgRole`) + new `getAvailabilityForPlayers` function, but tests still used old `role: "coach"` shape

Fixes:
- Regenerated `package-lock.json` cleanly (`rm -rf node_modules package-lock.json && npm install`)
- `eslint-disable-next-line` for the intentional patterns
- Updated `api-players.test.ts` and `api-availability.test.ts` to match current route internals, added missing `appendAuditLog` / `verifyCsrfOrigin` / `getPlayerByDiscordId` mocks
- All 118 tests passing, lint 0 errors

### Cleanup
- Archived 5 stale top-level markdown files (`AGENTS.md`, `PROJECT_SUMMARY.md`, `TEAM_HUB_SETUP.md`, `TEAM_MESSAGE.md`, `IMPerfect_Session_Report_2026-03-31.md`) and `IMP docs/` into `docs/archive/`

## Features Shipped (9)

### 1. Announcements → Discord
POST `/api/announcements` now fires a Discord embed to the appropriate team channel(s) based on `target_audience`:
- `ALL` / `PLAYERS` → IMPerfect + Shadows + Echoes channels
- Division-specific → just that channel
- `COACHES` / `MANAGERS` → main IMPerfect channel
- Pinned announcements get brand color + pin emoji
- Fire-and-forget (per convention), body truncated at 1800 chars

### 2. Scrim Decline + Feedback
External teams had no way to know why their scrim application was rejected.
- Migration: `scrim_applications.decline_reason TEXT`
- PATCH route accepts `decline_reason`, trims + 500-char cap
- Discord embed to `DISCORD_CHANNEL_SCRIMS` with reason + reviewer
- Panel: Decline button prompts for reason; declined apps show reason inline in red

### 3. VOD Library
VOD Review blocks were only visible inline in the schedule grid edit mode.
- New `/team-hub/vod` page listing all `VOD_REVIEW` blocks across divisions, sorted by most recent week
- Division filter tabs when multiple divisions exist
- Auto-detects URLs in notes → clickable pill links (plus inline linkification in body text)
- HEAD_COACH+ can add/edit notes inline without leaving the page
- Visible to all authenticated users (VODs are a learning tool for the whole team)
- New GET `/api/vod-reviews` + `getAllVodReviews()` in `db.ts`

### 4. Match History + Head-to-Head
- New `/team-hub/history` page
- Top stats row: overall W-L-D, win rate, total matches, unique opponents
- H2H section: per-opponent W-L-D, win rate, last played, sorted by total matches
- All Matches feed sorted by recency with score + result chip
- Game filter (All / OW2 / MR), division filter
- Read-only, all team members can view

### 5. Community Rejection Appeal Path
The duplicate guard already excluded `declined` status — captains just didn't know they could reapply, or why they were declined.
- Migration: `community_teams.decline_reason TEXT`
- PATCH accepts `decline_reason`, trims + 500-char cap
- Discord embed to `DISCORD_CHANNEL_COMMUNITY` with reason + reapply link to `imperfectgg.com/en/community/join`
- Panel shows reason inline in red on declined teams

### 6. Player Stats Filters + CSV Export
- FilterGroup pill buttons for Division and Role (auto-hidden if only 1 option)
- Player count updates to "Showing N of M"
- CSV export: player, division, role, rank, win rate, KDA, matches, data source, last updated, top 3 heroes with per-hero WR/KDA/matches. Date-stamped filename

### 7. Tournament Prep Section
Tournament data lived only in Management dashboard — invisible to the team that needed to prep for them.
- New `/team-hub/tournaments` page
- Upcoming & Ongoing events sorted by start date, plus last 10 completed results
- Countdown chip: `Today` / `Tomorrow` / `In N days` — highlighted when ≤7 days out
- Separate registration deadline countdown
- Prize pool + entry fee shown when set
- HEAD_COACH+ can add/edit prep notes inline (study targets, opponent scouts, comp plans, VOD links)
- New narrow PATCH `/api/team-hub/tournaments/[id]` scoped to notes only

### 8. Lineup Submissions (MVP)
The `submitLineup` / `approveLineups` permissions had zero backing infrastructure.
- Migration: `lineups` table with `scrim_id` FK, JSONB `slots`, status CHECK constraint, unique index on `scrim_id`
- `src/lib/lineup-rules.ts`: game-aware rules — OW2 (1 Tank / 2 DPS / 2 Support) vs MR (1 Vanguard / 2 Duelists / 2 Strategists / 1 Flex)
- GET `/api/lineups?scrim_id=` (authenticated)
- POST `/api/lineups` (HEAD_COACH+ or CAPTAIN) — upserts by scrim_id, validates slot count + role keys per `scrim.game`, rejects duplicate players, writes audit entry, fires Discord embed on submit
- `/team-hub/lineups/[scrimId]` page — role-grouped slot dropdowns, roster filtered to scrim's division, used-player dedupe with "already used" hint, 2000-char notes
- "Lineup" button added to every scrim card in ScrimsPanel

### 9. Lineup Approval + Hero Picks
- POST `/api/lineups/[id]/approve` (ORG_ADMIN+ gated), writes audit entry, fires green Discord embed
- Builder shows "Pending approval" banner with Approve button when status=submitted and viewer has `approveLineups` permission
- Approved lineups show "✓ Lineup approved" banner
- Optional `hero: string | null` on each LineupSlot (40-char text input — deliberately no catalog, avoids stale hero rosters across patches)
- Hero shows in italics next to `@-mentions` in both submit and approve Discord embeds
